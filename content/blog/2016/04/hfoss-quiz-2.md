In the [Humanitarian Free and Open Source Software Development (HFOSS)](https://hfoss-ritjoe.rhcloud.com/) course at the [Rochester Institute of Technology](https://www.rit.edu/), quizzes are in the form of blog posts submitted during the class period. The room stays quiet, but it is an open IRC quiz, so many of the students collaborated with each other in [#rit-foss](https://webchat.freenode.net/?channels=rit-foss) on freenode for the quiz.

This post is my quiz submission for the Spring 2016 semester [Quiz #2](https://hfoss-ritjoe.rhcloud.com/quiz/quiz2).

## HFOSS Spring 2016, Quiz #2

#### Expand each of the following acronyms.

1. IRC: **[Internet Relay Chat](https://en.wikipedia.org/wiki/Internet_Relay_Chat)**
2. FOSS: **[Free and Open Source Software](https://en.wikipedia.org/wiki/Free_and_open-source_software)**
3. OLPC: **[One Laptop per Child](https://en.wikipedia.org/wiki/One_Laptop_per_Child)**
4. DVCS: **[Distributed version control system](https://en.wikipedia.org/wiki/Distributed_version_control)**
5. FSF: **[Free Software Foundation](https://en.wikipedia.org/wiki/Free_Software_Foundation)**
6. PR: **[Pull Request](https://en.wikipedia.org/wiki/Distributed_version_control#Pull_requests)**

#### What is the short, two-letter name for the OLPC computers used in the final project for this class?

- [XO laptops](https://en.wikipedia.org/wiki/OLPC_XO)

#### What is the one-word name for the interface used in the OLPC computers?

- [Sugar](https://en.wikipedia.org/wiki/Sugar_%28software%29) (SoaS)

#### What is the grade level we are targeting our OLPC applications for?

- [4th grade students](https://hfoss-ritjoe.rhcloud.com/static/decks/nysp12cclsmath-grade4only.pdf)

#### Briefly define the following instructional theories, giving the role of the instructor in each.

- Didactic: **Lecturing; instructor speaks to class about topic**
- Dialectic: **Similar to the Socratic seminar; discussion-based instructional teaching with interactions between the instructor and the class**
- Constructivist: **Teaching style that combines experiences with teaching to create a learning experience that lends power to the learner; the instructor acts as a facilitator rather than a traditional lecturer**

#### Several elements are combined in different ways to form the Creative Commons licenses. Match each shorthand given in the list with the description of that license element below.

1. **NC**: [Non-Commercial](https://creativecommons.org/licenses/by-nc/4.0); You may not use the work for commercial purposes.
2. **SA**: [Share-alike](https://creativecommons.org/licenses/by-sa/4.0); You must convey the same rights "downstream" that were conveyed to you by "upstream".
3. **ND**: [No Derivatives](https://creativecommons.org/licenses/by-nd/4.0); You may not make changes to the work.
4. **BY**: [Attribution](https://creativecommons.org/licenses/by/4.0); You must attribute the contributions of the original or upstream creators of the work.

#### The presence of which license elements make a license "non-free" according to the FSF?

- (1) [Non-Commercial](https://creativecommons.org/licenses/by-nc/4.0); You may not use the work for commercial purposes.
- (3) [No Derivatives](https://creativecommons.org/licenses/by-nd/4.0); You may not make changes to the work.

#### Which license element is a copyleft? (give the letter, 1 pt)

- (2) [Share-alike](https://creativecommons.org/licenses/by-sa/4.0); You must convey the same rights "downstream" that were conveyed to you by "upstream".

#### Name two projects that distribute a body of non-software, free culture data, and briefly name or describe the kind of data.

1. [Fedora Project Documentation](https://docs.fedoraproject.org/en-US/index.html) (CC-BY-SA 3.0): Provides documentation and instructions on using the Fedora operating system and how to achieve certain tasks using it
2. [ManyBooks](http://manybooks.net/categories/CCL) (mixed variety of CC licensed works): Provides downloads of freely licensed literature, novels, and books

#### List or describe “[the four R’s](http://opencontent.org/blog/archives/1123)” as a shorthand for the freedoms attached to software for it to be considered “free and open source”.

1. Read
2. Run
3. Revise
4. Redistribute
